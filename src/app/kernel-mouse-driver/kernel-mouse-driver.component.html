Kernel-Level Mouse mini-driver

The driver doesn't reimplement the entire mouse processing logic. Instead, it finds and calls the existing kernel routine (MouseClassServiceCallback) that the standard Windows mouse driver uses to process input, feeding it custom-made input data. This is precisely how it bypasses higher-level APIs – it talks directly to the lower-level component responsible for handling raw mouse data.

Note: This driver is presented as a technical demonstration of kernel-level programming and input manipulation techniques. Primarily for educational purposes.
